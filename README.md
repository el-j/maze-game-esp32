# Haptic Tilt-Maze Console 🎮

[![CI](https://github.com/el-j/maze-game-esp32/actions/workflows/ci.yml/badge.svg)](https://github.com/el-j/maze-game-esp32/actions/workflows/ci.yml)
[![Pages](https://github.com/el-j/maze-game-esp32/actions/workflows/pages.yml/badge.svg)](https://el-j.github.io/maze-game-esp32/)

> **Live demo →** <https://el-j.github.io/maze-game-esp32/>  
> Play the exact same C++ game physics in your browser via WebAssembly — no hardware needed.

A handheld, battery-powered maze game built on an **ESP32**.  
Tilt the board to roll a pixel through maze corridors.  
Crash into walls and feel the rumble. Complete all levels and hear the fanfare.

---

## Hardware Stack

| Component | Part |
|---|---|
| Brain | ESP32 DevKit V1 |
| Motion sensor | GY-521 (MPU6050) |
| Display | Raw 8×8 LED matrix (e.g. 1088AS) |
| Haptics | Small DC motor + NPN transistor |
| Audio | Piezo buzzer |
| Input | Tactile push button |
| Power | 18650 battery shield |

## Quick Start

```bash
# Install PlatformIO (once)
pip install platformio

# Build + flash C++ firmware
make build
make upload

# Watch the Serial Monitor (115200 baud)
make monitor

# Run C++ host unit tests (no hardware needed)
make test

# ── Rust firmware ──────────────────────────────────────
# Install Espressif Rust toolchain (once)
cargo install espup && espup install && source ~/export-esp.sh

# Run Rust host tests (no hardware, no special toolchain)
make rust-test

# Cross-compile + flash Rust firmware
make rust-build
make rust-flash

# ── Browser demo ───────────────────────────────────────
# Requires Emscripten SDK (https://emscripten.org)
make wasm        # → webapp/public/game.{js,wasm}
make webapp-dev  # → http://localhost:3000
```

## Game Flow

```
TITLE ──(button)──► PLAYING ──(crash, lives>0)──► CRASHED ──► PLAYING
                       │                                          ▲
                       │                                          │
                       ├──(crash, lives=0)──────────► GAMEOVER ──┘ (button → TITLE)
                       │
                       ├──(goal, more levels)──────► next level
                       │
                       └──(goal, last level)───────► VICTORY (button → TITLE)
```

## Sound Design

Four non-blocking melodies play automatically via the piezo buzzer:

| Event | Sound |
|---|---|
| Boot | G–C–E–G ascending jingle |
| Wall crash | A–E descending thump |
| Level clear | C–E–G–C arpeggio |
| All levels done | Full 8-note victory fanfare |

## C++ vs Rust – Language Comparison

Both implementations run the exact same game logic on the exact same ESP32 chip. The landing page at <https://el-j.github.io/maze-game-esp32/#comparison> shows live binary-size measurements generated by CI on every push to `main`.

| | C++ / Arduino | Rust / esp-hal |
|---|---|---|
| Framework | PlatformIO + Arduino | esp-hal (bare-metal, `no_std`) |
| Language standard | C++17 | Rust 2021 edition |
| Opt level | `-Os` | `opt-level = "s"` + LTO + strip |
| Unit tests | PlatformIO native (Unity) | `cargo test` (host) |
| WASM target | Emscripten | — (C++ only for browser demo) |
| Binary sizes | See live comparison →  | ← <https://el-j.github.io/maze-game-esp32/#comparison> |

## CI

Every push and pull request to `main` triggers:

1. **`firmware-cpp`** – PlatformIO compile (release + debug)
2. **`firmware-cpp-tests`** – native Unity unit tests on the host
3. **`firmware-rust-tests`** – `cargo test --lib` (host, stable Rust)
4. **`firmware-rust-esp32`** – Rust cross-compile for `xtensa-esp32-none-elf` *(main only)*
5. **`webapp`** – 84 Jest tests + TypeScript type-check
6. **`size-comparison`** – extracts binary sizes → `comparison.json` *(main only)*
7. **`pages`** – Emscripten WASM build + `next build` → GitHub Pages *(main only)*
